Manim - An introduction

Local development environment setup

Manim is a custom Python library for mathematical visualizations. It has two versions, the first version is maintained by Grant Sanderson himself (the acclaimed Stanford professor who goes by the username 3blue1brown on YouTube). And another one, a fork of the original, called the Manim community edition which enjoys more contributions and has better documentation and support (in terms of attentiveness to PRs and such). For this series, I will use the community edition to help maintain this document’s relevance and, possibly, contribute to the open-source community project myself.

Installation

Optional - Create an isolated Python environment

Conda is an open-source package management and environment management system that is primarily used for Python, but it can also handle other programming languages like R. It’s designed to make it easy to install, run, and manage libraries and software dependencies in isolated environments, ensuring that different projects don’t interfere with each other. Alternatives to conda include pip (Python Package Installer), virtualenv (venv), and their merger Pipenv.

First, activate an environment (with your choice of environment and/or package manager from above). I will be using conda to activate a new dedicated environment as manim-playground using:

conda update conda # To, first, update your conda version.
Note

You need to deactivate any active conda environment before running this command.

Then:

conda create --name manim-playground

This will create a conda environment in ~/anaconda3/envs/manim-playground. Activate it with:

conda activate manim-playground

To deactivate the environment at any point, use:

conda deactivate

Manim installation

Here are the conda installation steps for Manim.

To install Manim we simply do:

conda install -c conda-forge manim

Jupyter Notebooks

To import Manim:

from manim import *
Note

This may trigger your IDE of choice, which is VS Code in my case, to download the ipykernel into the conda environment. This should be taken care of automatically. If not, installing Python into the environment should be straightforward. To avoid this in the future, instead of creating a blank conda environment we can clone from the base environment by:

conda create --name manim-playground --clone base

To see any output inside Jupyter Notebooks, we should also supply the magic command to invoke Manim (the animation engine). This magic command is its own installation – jupyter-manim. Note that only pip3 is available as a package manager for jupyter-manim but that’s ok becuase pip3 respects conda environments. When we run pip3 install <package>, pip installs the package and its dependencies in the site-packages directory of that specific, active conda environment, just as it would in any standard Python environment. So issue:

pip3 install jupyter-manim

The Grant Sanderson setup

Grant himself defines a custom script that hooks into a local Python file defining the entire scene, and parses the script looking for certain leading comments. It then runs the script only up to that comment, so that the scene pauses at the last step, at the part of the code he’s still working on. He then uses an IDE shortcut to execute just that single code block (clearly demarcated with comments) in a Python interpreter, which renders the relevant portion of the scene to the output window. In this way, he can incrementally test parts of the scene (mimicking a Jupyter notebook setup)

More on this specific setup can be found in the official Manim YoutUbe tutorial.

Scene object

We can now draw out first Scene. A guiding principle behind Manim’s design is that everything should be transformable into everything else. We can transform shapes into other shapes and even transform text elements into shapes (we will see an example of this shortly).

Square to Circle Transformation

%%manim -ql -v WARNING SquareToCircle

class SquareToCircle(Scene):
    def construct(self):
        # Obbligatory Hello World
        message = Text("Hello, World!")

        # Assing simple geometry
        circle = Circle()
        square = Square()
        self.add(circle)
        self.add(square)

        # Animating text
        self.play(
            Write(message, run_time=3)
        )
UsageError: Line magic function `%%manim` not found.

As we can see from above: everything in Manim derives from the Scene class which offers a constructor method. In this constructor metho we define every object in ourscene, every transformation, and, in-short, run the entire sequence of actions in our scene.
